THE 92 — FEATURE BACKLOG
========================
Ordered by natural build sequence. Each phase depends on the one before it.
Data must exist before UI, core mechanics before social, social before matchday.


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
PHASE 0 — SEED DATA
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Nothing works without real clubs, grounds, and stats. Do this first.

T-001  Import all 92 clubs and grounds
       Import all 92 current Football League / Premier League clubs
       with their ground as a management command or fixture.
       Minimum fields per team: name, slug, league_level, is_current_92=True.
       Minimum fields per ground: name, slug, team, town_or_city, postcode.
       Source: curated CSV built from Wikipedia's "The 92 Club" article.

T-002  Populate ground stats
       For each ground, add: capacity, opened_year, address.
       Source: Wikipedia infoboxes (can be partially automated via the
       Wikipedia API — search "<ground name> football ground").
       Target fields already exist on the Ground model.

T-003  Add latitude/longitude to Ground model
       Add lat/lng FloatFields (nullable) to Ground for map support later.
       Geocode each ground postcode using the free Nominatim/OpenStreetMap API
       via a management command (respect rate limits: 1 req/sec).
       Migration required.

T-004  Add ground pitch dimensions to Ground model
       Add length_m and width_m (PositiveIntegerField, nullable).
       Source: Wikipedia infoboxes. Most top-flight grounds have these.
       Migration required.

T-005  Upload team logos
       Source club crests from a consistent source (e.g. Wikipedia SVGs or
       Football-Data.org). Store in media/team-logos/ via the existing
       Team.logo ImageField.

T-006  Historical teams fixture
       Add a small fixture of notable non-92 teams (e.g. national teams,
       Scottish Premier League clubs) so favourite_team on signup has
       meaningful options beyond the 92.

The heart of the product. Users track which grounds they have visited.

T-013  Visit model
       New model: Visit(user, ground, visit_type, visited_on, notes).
       visit_type choices: VISITED, WANT_TO_GO, HISTORIC.
       visited_on: DateField, nullable (historic visits may lack a precise date).
       A user can have multiple visits to the same ground (return trips).
       One WANT_TO_GO per user per ground (enforce via unique_together or clean).
       Migration required.

T-014  Mark ground as visited — UI and view
       "Claim ground" button on ground detail page (red, primary action).
       POST to a simple view that creates a Visit(type=VISITED, visited_on=today).
       Redirect back to the ground detail. Show confirmation state on the button.

T-015  Add to want-to-go list — UI and view
       Secondary action button on ground detail: "Want to go".
       Creates Visit(type=WANT_TO_GO). Toggle off if already added.

T-016  Add historic visit — UI and view
       Separate form on ground detail: "Add a historic visit" with a date picker.
       Creates Visit(type=HISTORIC, visited_on=<date>). Allows multiple entries.

T-017  Visited state on ground cards and detail
       If the logged-in user has visited a ground, show a "Visited" pill
       (pitch/lime colour) on the card and detail page.
       Use select_related and a prefetched visit queryset for efficiency.

T-018  User visit count
       Replace the hardcoded visited_count=57 in the home view with a real
       queryset: Visit.objects.filter(user=request.user, visit_type=VISITED)
       .values('ground').distinct().count()

T-019  Tests for Visit model and collecting views
       Cover: creating a visit, duplicate WANT_TO_GO prevention,
       visited state rendering on cards, unauthenticated access redirects.

DATA SOURCES REFERENCE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Clubs and ground names
  Wikipedia — "List of Football League clubs" and "The 92 Club" articles.
  Curated CSV is the most reliable starting point; Wikipedia API for bulk stats.

Capacity and opened year
  Wikipedia infoboxes per club. API endpoint:
  https://en.wikipedia.org/api/rest_v1/page/summary/<Club_Name_FC>
  or use the MediaWiki parse API for full infobox data.

Pitch dimensions
  Wikipedia infoboxes. Not available for all grounds.

Postcodes and addresses
  The Football Ground Guide (footymad.net) has every ground address.
  Alternatively, build from Wikipedia + manual correction.

Geocoding (lat/lng from postcode)
  Nominatim / OpenStreetMap:
  https://nominatim.openstreetmap.org/search?postalcode=<postcode>&country=GB&format=json
  Free, no key required, 1 request/second policy.

Team colours
  Wikipedia or manually curated. See also Football Colour API (unofficial).

Fixtures and results
  Football-Data.org — free tier covers Premier League + Championship + EFL.
  Register for a free API key. Python client available.
  https://www.football-data.org/documentation/quickstart

Club crests / logos
  Wikipedia Commons has SVG crests for most EFL clubs under free licences.
  Verify licence per club before use.

Away fan information (parking, pubs, transport)
  The Football Ground Guide (footymad.net) — community-sourced, comprehensive.
  Can be used for initial seed content with attribution; plan to replace
  with community contributions over time.
